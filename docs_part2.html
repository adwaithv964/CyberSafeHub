4.5 Database Design

CyberSafeHub uses MongoDB Atlas as its primary database, leveraging the NoSQL document model for flexibility in handling varying data structures across tools. The database is named cybersafehub and contains the following collections:

Collection Document Count (est.) Purpose Schema File
admins Small (5–20) Admin user records with roles Admin.js
vaultitems Medium (100–10,000) Encrypted user credentials VaultItem.js
threatlogs Large (grows continuously) Malware scan audit records ThreatLog.js
jobs Medium (rolling window) File conversion job queue Job.js
platformstats Small (daily records) Aggregated platform analytics PlatformStat.js

Design Decisions:

  • User profile data is stored in Firebase (not MongoDB) to avoid storing authentication credentials in a separate database.
  • Vault passwords are encrypted with CryptoJS AES before insertion — MongoDB stores only ciphertext, never plaintext passwords.
  • Threat logs are append-only; no deletion is performed to maintain audit integrity.
  • The jobs collection uses a status field (pending/processing/done/error) to support asynchronous file conversion via worker processes.

4.7 User Interface Design

CyberSafeHub's interface follows a dark-mode cyberpunk aesthetic with a professional, accessible design system built on TailwindCSS. Key design decisions:

  • Colour Palette: Deep navy (#0f172a) backgrounds, electric cyan (#22d3ee) and purple (#8b5cf6) accents, white (#f1f5f9) text on dark surfaces.
  • Typography: System font stack with Inter-like fallbacks. Bold headings with tracking for cyberpunk feel; regular weight for body text.
  • Component Library: Custom React components — Button.jsx, Card.jsx, Input.jsx, ThemedModal.jsx — ensure consistent styling across the platform.
  • Animations: Framer Motion powers page transitions, card hover effects, modal entry/exit, and the animated blob backgrounds. All animations run at 60fps.
  • Three.js Scene: The dashboard features a 3D animated cybersecurity scene (floating nodes and connections) rendered via @react-three/fiber, creating an immersive first impression.
  • Sidebar Navigation: A persistent, collapsible left sidebar provides access to all major modules. On mobile, it collapses to a bottom tab bar via responsive breakpoints.
  • Glassmorphism Cards: Tool cards use a frosted-glass effect (backdrop-blur, semi-transparent backgrounds) for a modern, premium look.
  • Responsive Layout: CSS Grid and Flexbox layouts adapt from 1-column (mobile) to 3-column (desktop) grids. All tools are fully functional on 375px–2560px screen widths.

4.8 Input Design

Input design in CyberSafeHub prioritises safety, clarity, and accessibility. All inputs include real-time validation with visual feedback.

Module Input Type Fields / Controls Validation

AI Chat Text Input Chat message Non-empty, max 2000 chars, shift+enter newline
Admin — User Search Search Input Email or UID search Debounced search, min 3 characters

Common Input Design Principles Applied:

  • All password fields include a show/hide toggle button.
  • File upload inputs feature drag-and-drop zones with visual feedback.
  • Submit buttons are disabled while processing to prevent duplicate submissions.
  • Error messages appear inline below the relevant field in red text.
  • All text inputs use autocomplete="off" for sensitive fields (vault).
