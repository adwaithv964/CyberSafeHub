1. INTRODUCTION

1.1 Purpose

The purpose of this document is to provide a comprehensive technical and functional overview of CyberSafeHub — a full-stack web application designed to bridge the critical gap between cybersecurity knowledge and everyday digital users. In an era where cyber threats such as phishing, ransomware, identity theft, and data breaches have become alarmingly common, there exists an urgent need for a centralised, accessible, and interactive platform that equips users with the tools, education, and awareness necessary to protect themselves online.

CyberSafeHub serves as a one-stop digital security hub that combines:

  • Practical security utilities — tools to scan networks, wash metadata, detect malware, manage passwords, and perform OSINT lookups.
  • AI-powered intelligence — Gemini API integration for code auditing, privacy policy decoding, and an interactive cyber assistant.
  • Cybersecurity education — structured academy modules, phishing simulations, encryption visualisers, and an RPG-style incident response simulator.
  • Emergency response — guided step-by-step recovery plans for hacked accounts, stolen devices, and ransomware attacks.

This document follows a structured Software Development Life Cycle (SDLC) approach, covering system study, requirements specification, design, development, testing, implementation, and future roadmap — providing a thorough reference for developers, evaluators, and project stakeholders.

1.2 Scope

The scope of CyberSafeHub encompasses the design, development, and deployment of a responsive web application accessible via modern browsers on desktop and mobile devices. The system is targeted at a broad audience — from general internet users seeking basic digital safety knowledge, to IT students and professionals requiring advanced security utilities.

Within Scope:

  • User registration, login (email/password and Google OAuth), multi-factor authentication (TOTP), and profile management via Firebase Authentication.
  • A centralised dashboard providing navigation to all modules with real-time status indicators.
  • Security tools: Metadata Washer, WiFi Radar (network scanner), Username Detective (OSINT), Code Security Auditor (AI), Privacy Policy Decoder (AI), Steganography Studio, Password Vault, Password Cracker (educational), Malware Scanner (ClamAV), and Network Tools.
  • Educational academy with structured modules, phishing trainer, encryption visualiser, and an AI-powered incident response RPG game.
  • File conversion utility supporting 30+ format conversions (PDF, DOCX, images, audio/video) via FFmpeg and LibreOffice.
  • Emergency guides panel for crisis response situations.
  • Digital Privacy checklist and best practices guide.
  • Admin panel with user management, analytics dashboard, and threat log monitoring.
  • REST API backend (Node.js/Express) with JWT-equivalent Firebase token authentication and MongoDB Atlas data storage.
  • Real-time communication via Socket.IO for scan progress updates.
  • Deployment via Docker/Render (backend) and Vercel (frontend).

Out of Scope:

  • Native mobile applications (iOS/Android) — the platform is web-based only.
  • Real-time Nmap-based deep port scanning (simulated in current version).
  • Paid subscription tiers or payment gateway integration.
  • Offline-first Progressive Web App capabilities (future enhancement).

2. SYSTEM STUDY AND ANALYSIS

2.1 Preliminary Investigation

The preliminary investigation was initiated by analysing the growing landscape of cyber threats affecting everyday users. According to global cybersecurity reports (IBM, Verizon DBIR 2024), over 68% of data breaches involve a human element — phishing, weak passwords, or misconfigured systems — indicating that user education and accessible security tools are the most impactful countermeasures.

An initial survey and market study revealed that:

  • Most cybersecurity tools are siloed — password managers are separate from network scanners, which are separate from educational platforms.
  • Enterprise-grade tools (Wireshark, Nmap, Burp Suite) have steep learning curves unsuitable for general users.
  • Free educational platforms lack hands-on, interactive components like simulations and AI-driven feedback.
  • No single platform combines OSINT lookup, file scanning, AI analysis, and educational modules under one authenticated interface.

The investigation concluded that building a unified, user-friendly cybersecurity hub with both educational and practical tools — integrated with modern AI capabilities — would address a genuine and significant market gap. The project was greenlit for development as a full-stack web application using React, Node.js, MongoDB, and Firebase.

2.2 Existing System

Several competing and complementary systems currently exist in the cybersecurity space:

System Category Limitations
Have I Been Pwned Breach lookup Only checks breaches; no tools or education
VirusTotal File/URL scanner No educational content; requires technical knowledge
1Password / Bitwarden Password manager No security tools or learning modules
Cybrary / TryHackMe Education platforms No practical security tools integrated
Shodan OSINT / Network scan Complex interface; no beginner guidance
ExifTool Metadata removal Command-line only; inaccessible to general users

Key shortcomings of existing systems: fragmentation, lack of integrated AI assistance, no gamified learning, no emergency response guidance, and complex interfaces not suitable for non-technical users.

2.3 Proposed System

CyberSafeHub is proposed as a unified, AI-augmented cybersecurity platform that consolidates the functionalities of multiple tools into one authenticated, user-friendly environment. The system leverages modern web technologies to deliver:

  • A clean, dark-mode React UI with smooth Framer Motion animations and TailwindCSS styling.
  • A secure Node.js/Express backend exposing RESTful APIs protected by Firebase token verification.
  • MongoDB Atlas for scalable, cloud-hosted NoSQL data storage.
  • Google Gemini API for AI-powered analysis, chat, and simulation.
  • ClamAV integration for real malware detection on uploaded files.
  • An admin panel for platform management with user analytics and threat monitoring.

2.3.1 Advantages of the Proposed System

  • All-in-One Platform: Eliminates the need for multiple separate tools by providing scanning, education, AI assistance, and emergency response in one place.
  • Beginner-Friendly Design: Clean visual interface with guided tooltips, plain-English AI explanations, and step-by-step emergency guides lower the barrier to entry.
  • AI-Powered Intelligence: Integration with Google Gemini API enables real-time code auditing, legal document decoding, chat-based Q&A, and dynamic game mastering — capabilities absent from all existing tools.
  • Gamified Education: The Incident Response Simulator and Phishing Trainer make security learning engaging and memorable, significantly improving knowledge retention.
  • Real Security Tools: ClamAV malware scanning, ARP network scanning, EXIF metadata removal, and steganography are genuine working utilities — not simulations.
  • Secure Authentication: Firebase Auth with TOTP MFA ensures strong identity verification for all users.
  • Scalability: MongoDB Atlas and cloud deployment (Render + Vercel) allow the platform to scale horizontally without infrastructure changes.
  • Privacy First: No permanent storage of scanned code or legal documents; vault items are encrypted before storage.
  • Cross-Platform Accessibility: Responsive design ensures full functionality on desktops, tablets, and mobile browsers.
  • Open and Extensible: Modular React component architecture and RESTful API design allow easy addition of new tools and features.

2.4 Module Description

CyberSafeHub is organised into the following primary modules:

Module 1 — Authentication & User Management

Handles all identity operations including registration, login (email/password + Google OAuth), multi-factor authentication with TOTP (Time-based One-Time Passwords), password reset, profile updates, and account deletion. Implemented via Firebase Authentication with custom React context (AuthContext). Admin roles are stored in MongoDB with Firebase Admin SDK for server-side verification.

Module 2 — Dashboard

The central landing page after login. Displays a personalised greeting, animated 3D cyberpunk scene (Three.js), quick-access navigation cards for all major tools, and a security posture summary. Background blobs (animated SVG shapes) provide an immersive visual environment.

Module 3 — Cyber Tools

A collection of practical security utilities:

  • Metadata Washer: Strips EXIF/GPS/camera metadata from JPG, PNG, WebP, HEIC images using the Sharp library (backend) and ExifJS (client-side preview).
  • WiFi Radar: Performs ARP table sweeps and ping scans on the local network to enumerate connected devices, identify vendor by MAC address, and flag anomalies.
  • Username Detective: Checks username availability across 50+ social media and online platforms (GitHub, Instagram, Reddit, Twitter, etc.) by proxying HTTP requests through the backend.
  • Code Security Auditor: Accepts source code in any language; sends it to the Gemini API with a security-focused system prompt and returns a structured vulnerability report.
  • Privacy Policy Decoder: Accepts legal text (up to 15,000 chars); AI returns a plain-English summary with a Privacy Score (0–10) and identified red flags.
  • Secure File Converter: Converts files between 30+ formats including PDF↔DOCX, image format conversion, audio/video transcoding via FFmpeg and LibreOffice.

Module 4 — Security Scanners

  • Malware Scanner: Accepts file uploads (up to 50MB), forwards to ClamAV via the clamscan Node.js library, returns a clean/infected verdict with threat name.
  • Network Tool: Retrieves the user's public IP, performs DNS leak tests, checks open ports, and retrieves geolocation data for IP addresses.
  • Health Check: An interactive checklist evaluating the user's current device/account security posture with a final security score.

Module 5 — Password Management

  • Password Vault: Stores encrypted credentials (website, username, password, notes) in MongoDB. Uses AES encryption via crypto-js before persistence. Supports CRUD, copy-to-clipboard, and password strength indicator.
  • Password Cracker (educational): Visual brute-force simulator showing real-time character permutations and estimated crack time for common/custom passwords.

Module 6 — Cyber Academy

  • Learning Modules: Structured educational content covering network security, cryptography, social engineering, safe browsing, and more.
  • Phishing Trainer: Simulates a realistic email inbox with genuine-looking phishing emails for the user to identify.
  • Encryption Visualizer: Interactive step-by-step demonstration of Caesar cipher, AES, and RSA encryption algorithms.
  • Incident Response Simulator: AI-driven RPG game (Gemini API) where the player acts as a sysadmin handling live cyber attacks — Ransomware, DDoS, SQL Injection, Insider Threats. Dynamic scenarios with consequence-based progression.

Module 7 — AI Cyber Assistant

A chat interface powered by the Gemini API providing context-aware answers to cybersecurity questions. The assistant maintains conversation history within a session for coherent multi-turn dialogue.

Module 8 — Emergency & Privacy

  • Emergency Guides: Guided recovery wizards for Stolen Device, Hacked Account, Ransomware Attack, and Identity Theft scenarios.
  • Emergency Contacts: Local-storage–based trusted contacts list for quick access during crises.
  • Digital Privacy: Comprehensive privacy checklist and best practice guides covering browsers, apps, social media, and network settings.

Module 9 — Admin Panel

Accessible only to verified administrators. Provides: user management (list, roles, disable accounts), analytics dashboard (total users, scan counts, threat logs), platform statistics, and activity logs. Backend routes are protected by the adminAuth middleware using Firebase Admin SDK token verification.

Module 10 — Steganography Studio

Allows users to hide (encode) secret text messages inside PNG images and extract (decode) hidden messages from images. Implemented using pixel-level LSB (Least Significant Bit) steganography in a client-side canvas algorithm.

Module 11 — Cyber News

Aggregates the latest cybersecurity news from RSS feeds and external APIs, displaying formatted articles with thumbnails, source attribution, and category tags.

2.5 Feasibility Study

2.5.1 Technical Feasibility

The technical feasibility of CyberSafeHub is highly positive. All core technologies used — React, Node.js, Express, MongoDB, and Firebase — are mature, well-documented, open-source frameworks with large developer communities and proven production-grade stability.

  • Frontend: React 18 with Vite delivers fast HMR (Hot Module Replacement) for development and optimised production bundles. TailwindCSS and Framer Motion provide responsive, animated UIs without heavy overhead.
  • Backend: Node.js + Express is a lightweight, non-blocking runtime ideal for I/O-heavy operations like file scanning and network probing. Socket.IO supports real-time bidirectional communication for scan progress.
  • Database: MongoDB Atlas is a fully managed cloud database that eliminates infrastructure concerns. Mongoose ODM simplifies schema definition and validation.
  • AI Integration: The Google Gemini API (REST) is straightforward to integrate via HTTP requests; no complex ML infrastructure is required on the server side.
  • Security Tools: ClamAV is a well-maintained open-source antivirus engine. The clamscan npm package provides a clean Node.js interface. ARP/ping network scanning uses OS-level system calls wrapped in local-devices library.
  • Deployment: Docker containerisation ensures environment consistency. Render (backend) and Vercel (frontend) provide zero-configuration CI/CD pipelines.
✅ Conclusion: All required technologies are proven, available, and within the team's skill set. Technical feasibility is CONFIRMED.

2.5.2 Operational Feasibility

CyberSafeHub is designed with operational ease as a core principle. The platform requires minimal user training due to its intuitive, icon-driven navigation and contextual help text throughout the interface. Key operational considerations:

  • User Adoption: The clean, dark-mode UI with progressive disclosure (simple overview → detailed results) reduces cognitive load. Users familiar with basic web apps will navigate without training.
  • Admin Operations: The admin dashboard provides all management functions (user control, log viewing, statistics) via a visual interface, eliminating need for direct database access.
  • Maintenance Overhead: Serverless Firebase Auth, managed MongoDB Atlas, and automated deployments via Render/Vercel reduce operational burden to near zero for standard operations.
  • Error Handling: The backend implements graceful error handling with user-friendly error messages; the frontend displays toast notifications for all success/failure states.
✅ Conclusion: The system is operationally straightforward and requires minimal support effort. Operational feasibility is CONFIRMED.

2.5.4 Behavioural Feasibility

User behavioural studies and UX design principles guided the interface design of CyberSafeHub:

  • Motivation: The growing awareness of online threats ensures users are motivated to engage with security tools. High-profile breaches (Facebook, LinkedIn, Aadhaar) have increased public interest in personal cybersecurity.
  • Gamification: The Incident Response Simulator and Phishing Trainer leverage game mechanics (decisions, consequences, win/lose states) to maintain engagement and improve retention of security concepts.
  • Psychological Safety: All analysis (code audit, policy decoding) is done on-demand with no permanent data retention, addressing user privacy concerns about submitting sensitive data.
  • Progressive Disclosure: The platform presents simple results first with option to drill down into technical details, accommodating both novice and expert users.
  • Habit Formation: The Health Check periodic questionnaire and Digital Privacy checklist encourage regular security hygiene reviews.
✅ Conclusion: User behaviour aligns with the platform's offerings. Behavioural feasibility is CONFIRMED.

2.5.5 Legal Feasibility

Legal compliance has been carefully considered throughout the design of CyberSafeHub:

  • Open Source Compliance: All libraries used (React, Express, MongoDB, Firebase) are distributed under permissive licences (MIT, Apache 2.0). No GPL libraries are used in a manner that would affect proprietary code.
  • OSINT Ethics: The Username Detective checks publicly accessible URLs only and does not scrape or store personal information beyond a transient session result. This is consistent with legal OSINT practices.
  • Data Privacy: Firebase Auth handles credential storage, ensuring compliance with GDPR-equivalent standards (Firebase is GDPR compliant). User vault data is encrypted before storage.
  • ClamAV Usage: ClamAV is a GNU GPL-licensed open-source virus scanner. Its use for scanning user-uploaded files is fully within its intended legal scope.
  • AI API Terms: The Gemini API is used in compliance with Google's Terms of Service. No personal PII is transmitted in AI prompts; analysis is limited to code and text.
  • No Unauthorised Network Scanning: WiFi Radar only scans the local network of the server host. Users are advised to scan only networks they own or have authorisation to test.
  • Content Disclaimer: Educational hacking simulations (Phishing Trainer, Incident Response) include disclaimers clarifying educational intent only.
✅ Conclusion: The system operates within all applicable legal frameworks. Legal feasibility is CONFIRMED.
