The deployment architecture separates the frontend (static assets) from the backend (API server + binary tools) to maximise performance, scalability, and security.

Deployment Steps

  1. Push latest code to GitHub main branch.
  2. Vercel auto-detects changes, builds frontend with vite build, and deploys to CDN within ~2 minutes.

  4. Environment variables are set in Render's dashboard (not stored in code).
  5. MongoDB Atlas IP allowlist updated with Render's outbound IP if changed.
  6. Smoke test: visit production URL, login, run a sample scan, verify AI response.

7.5 Security

Security is a first-class requirement throughout CyberSafeHub's architecture. The following security measures are implemented at every layer:

Authentication Security

  • Firebase Authentication with email verification and bcrypt-equivalent password hashing (managed by Firebase).
  • TOTP-based Multi-Factor Authentication support for enhanced account protection.
  • JWT-equivalent Firebase ID tokens (RS256 signed, 1-hour expiry) used for all API authentication.
  • Google OAuth 2.0 as an alternative authentication pathway.

API Security

  • Helmet.js: Sets security headers (X-Frame-Options, X-XSS-Protection, Content-Security-Policy, etc.) on all API responses.
  • CORS: Configured to allow only the production Vercel domain and localhost (development). The Authorization header is explicitly whitelisted.
  • Rate Limiting: express-rate-limit restricts each IP to 100 requests per 15 minutes, preventing brute-force and DDoS attacks.
  • Input Sanitisation: xss-clean strips XSS payloads from request body/query. express-mongo-sanitize removes MongoDB operators from user input.
  • File Upload Restrictions: Multer enforces MIME type whitelists and a 50MB size limit on uploaded files.

Data Security

  • Vault passwords are encrypted with AES-256 (CryptoJS) using the user's Firebase UID as part of the key derivation, before MongoDB storage.
  • MongoDB Atlas encrypts data at rest (AES-256) and in transit (TLS 1.2+).
  • No sensitive API keys are exposed in the frontend bundle — AI calls requiring server-side keys use backend proxies.
  • Uploaded files (scan jobs, conversions) are deleted from the server after processing to prevent data accumulation.

7.6 Maintenance

CyberSafeHub's maintenance plan is divided into three categories:

Corrective Maintenance

Bug fixes and error resolutions triggered by user reports or monitoring alerts. Bug reports are tracked via GitHub Issues. Critical bugs (security vulnerabilities, authentication failures) are patched within 24 hours. Minor UI/UX bugs are batched into weekly patch releases.

Adaptive Maintenance

  • Dependency updates — npm audit and npm update run monthly to address security vulnerabilities in dependencies.
  • Firebase SDK updates — Firebase periodically deprecates older APIs; SDK updates are applied within 30 days of release.
  • ClamAV definition updates — freshclam is configured to update virus signatures daily.
  • Gemini API model upgrades — Prompt engineering tuning as new Gemini model versions are released.

Perfective Maintenance

  • Performance optimisations based on Lighthouse and real-user metrics.
  • UI/UX improvements from user feedback collected via in-app form.
  • New tool and feature additions as described in the Future Enhancement section.

10.1.1 — Authentication Context (src/contexts/AuthContext.jsx)

import { createContext, useContext, useEffect, useState } from 'react';
import { auth } from '../config/firebase';
import {
  onAuthStateChanged, signInWithEmailAndPassword,
  createUserWithEmailAndPassword, signInWithPopup,
  GoogleAuthProvider, signOut, updateProfile
} from 'firebase/auth';

const AuthContext = createContext();
export const useAuth = () => useContext(AuthContext);

export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const unsubscribe = onAuthStateChanged(auth, (currentUser) => {
      setUser(currentUser);
      setLoading(false);
    });
    return () => unsubscribe();
  }, []);

  const login = (email, password) =>
    signInWithEmailAndPassword(auth, email, password);

  const register = (email, password) =>
    createUserWithEmailAndPassword(auth, email, password);

  const googleLogin = () =>
    signInWithPopup(auth, new GoogleAuthProvider());

  const logout = () => signOut(auth);

  return (
    <AuthContext.Provider value={{ user, loading, login, register, googleLogin, logout }}>
      {!loading && children}
    </AuthContext.Provider>
  );
};

10.1.2 — Backend Auth Middleware (server/middleware/auth.js)

import admin from '../config/firebaseAdmin.js';

export const protect = async (req, res, next) => {
  const authHeader = req.headers.authorization;
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'No token provided' });
  }
  const token = authHeader.split(' ')[1];
  try {
    const decodedToken = await admin.auth().verifyIdToken(token);
    req.user = decodedToken;
    next();
  } catch (error) {
    console.error('Auth error:', error.message);
    res.status(401).json({ error: 'Invalid or expired token' });
  }
};

export const adminAuth = async (req, res, next) => {
  await protect(req, res, async () => {
    try {
      const adminDoc = await Admin.findOne({ uid: req.user.uid });
      if (!adminDoc) return res.status(403).json({ error: 'Admin access required' });
      req.admin = adminDoc;
      next();
    } catch (err) {
      res.status(500).json({ error: 'Admin verification failed' });
    }
  });
};

10.1.3 — Malware Scanner Route (server/index.js — /scan endpoint)

import NodeClam from 'clamscan';
import multer from 'multer';
import path from 'path';
import fs from 'fs';

const upload = multer({
  dest: 'uploads/',
  limits: { fileSize: 50 * 1024 * 1024 } // 50MB
});

const ClamScan = new NodeClam().init({
  removeInfected: false,
  quarantineInfected: false,
  scanLog: null,
  debugMode: false,
});

app.post('/scan', upload.single('file'), async (req, res) => {
  if (!req.file) return res.status(400).json({ error: 'No file uploaded' });
  const filePath = path.resolve(req.file.path);
  try {
    const clam = await ClamScan;
    const { isInfected, viruses } = await clam.scanFile(filePath);
    fs.unlink(filePath, () => {}); // clean up
    res.json({
      status: isInfected ? 'infected' : 'clean',
      threat: isInfected ? viruses[0] : null,
      file: req.file.originalname,
      size: req.file.size
    });
  } catch (err) {
    fs.unlink(filePath, () => {});
    res.status(500).json({ error: 'Scan failed', detail: err.message });
  }
});

10.1.5 — WiFi Radar Scan API (excerpt from server/index.js)

import localDevices from 'local-devices';
import { Resolver } from 'dns';
import lookup from 'oui';

app.get('/api/wifi-radar/scan', protect, async (req, res) => {
  try {
    const devices = await localDevices();
    const enriched = await Promise.all(devices.map(async (device) => {
      let vendor = 'Unknown';
      try { vendor = lookup(device.mac) || 'Unknown'; } catch {}
      let hostname = device.ip;
      try {
        const resolver = new Resolver();
        const names = await resolver.resolvePtr(device.ip);
        if (names.length) hostname = names[0];
      } catch {}
      return {
        ip: device.ip,
        mac: device.mac,
        vendor,
        hostname,
        riskLevel: vendor === 'Unknown' ? 'medium' : 'low'
      };
    }));
    res.json({ devices: enriched, scannedAt: new Date() });
  } catch (err) {
    res.status(500).json({ error: 'Network scan failed', detail: err.message });
  }
});
